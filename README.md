## Daily routine

1. **Code:** Write at least one C file per day (even small practice counts).
   - Follow the day-wise folder structure
   - Keep files well-commented

2. **Commit:** Push to GitHub with a meaningful message.
   - Example: `Day 2: Pointers Part 1 + memory basics`
   - Example: `Mini-project: Password Strength Rater v1`

3. **Track:** Update `notes/progress-log.md` with:
   - What you learned today
   - Minutes spent coding
   - Topics struggled with or mastered
   - Code files or resources used

4. **Reflect:** Once a week, review gaps and adjust next week's focus.
